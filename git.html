Git In One Sentence

GIT is a version control system designed to track the history of text files across multiple distributed instances, without the need for constant communication between instances, by representing files, their relationships, and their history as complete, raw data blobs stored in a compressed database indexed by statistically unique, content based, pseudo-random hashes and whose entire history and content can be accessed, from any arbitrary starting point, by means of recursive traversal of its node space, which is best represented in the form of a directed acyclic graph mapped, along with its content, in four dimensions.

Why People Find Git Confusing

  • Directed Acyclic Graphs
  • Recursive Traversal
  • Accurate diagram needs 4 dimensions
  • LOTS of incorrect information floating around
  • Distributed Structure
  • Trying to understand git in context of earlier source-control systems
  • Contains multiple trees
  • Contains multiple environments
  • Values speed and efficiency over ease of use
  • Command line syntax seems inconsistent

Hash Space

  • Git uses SHA1 hash
  • Content based
  • Waterfall, change one bit much of the hash changes
  • Pseudo-random
  • Statistically unique
  • 2160 possible hashes
  • That's big...
  • For simplicity, we've compressed the space into a normal 54 card deck
